❓ 使い方・ボタンの説明

    
テキストで指示するだけで、PCのAI（Claude Code）がファイル編集・コマンド実行など作業します。各ボタンの役割は以下のとおりです。


    
ヘッダー（最上段）

    
🔄アプリを更新。最新版のUIを取り込みます（表示が古いと感じたら）。

    
❓このヘルプを開きます。

    
⚙️設定。接続先URL・モデル・権限・Passkey・エージェント再起動など。

    
●接続状態の表示（稼働中／切断）。


    
Git バーの VS Code

    
⬡VS Code。選んだプロジェクトをブラウザ版 VS Code（vscode.dev）で開きます（タブレットや大きい画面で見たいとき向け）。


    
プロジェクト選択の行

    
▾プロジェクト選択。指示の対象フォルダを切り替えます（左右スワイプでも切替）。

    
🆕新しい会話。それまでの文脈を切ってまっさらから始めます（話が混ざったとき）。

    
🗂ジョブ履歴。実行中・完了した指示の一覧。タップで会話に復元できます。

    
🚀プロセス管理。npm run dev や pytest などの常駐起動とログ確認。

    
📋状況解説。gitの変更などをAIが人間語で説明します。

    
📅今日のまとめ。今日の作業をAIがレポートします。

    
📂ファイル一覧。プロジェクト内を辿って中身を確認できます。

    
＋プロジェクト追加。フォルダのパスを登録します。


    
Git 操作（変更の保存・共有）

    
Git は「作業の履歴を残して、ネット上（GitHub等）に保存・共有する」しくみです。下のボタンは選択中のプロジェクトに対して動きます。専門用語が分からなくても、下の順番どおり押せばOKです。

    
↓Pull（取り込み）。ネット上の最新を手元に取り込みます。作業を始める前に押すと、他のPCで進めた分が反映されます。

    
≡Status（状態確認）。今どのファイルを変えたか、一覧で確認できます。「何をいじったっけ？」を見たいとき。

    
✓Commit（保存）。変更内容に名前（メッセージ）を付けて手元の履歴に記録します。押すとメッセージ入力が出ます。区切りのいいところで押す「セーブ」のイメージ。変更は全部まとめて記録されます。

    
↑Push（送信）。Commit で記録した内容をネット上（GitHub等）に送って共有・バックアップします。Commit だけではネットに反映されません。最後に Push が必要です。

    
▶使う順番：① ↓Pull（最初に最新化）→ ② 作業（指示や編集）→ ③ ≡Status（確認）→ ④ ✓Commit（保存）→ ⑤ ↑Push（送信）。


    
入力エリア

    
⭐お気に入り。よく使う指示を登録してワンタップ送信。

    
📎添付。画像やファイルを添付して指示できます。

    
🔒金庫（Vault）。APIキー・パスワード・ニーモニック等の機密値を、チャット履歴・画面・ログに残さずAIとやり取りします。指紋か暗証番号で開錠。値はスマホ端末内だけに暗号化保存（PCには保存しません）。双方向に使えます↓

    
A→渡す（スマホ→PC）。🔒で値を登録（例: 名前 API_KEY）→ 開錠中に名前チップをタップすると本文に {{API_KEY}} が入ります（タップでコピーも可）。送信時にだけ実値へ置換され、履歴・ログには {{API_KEY}} の名前だけ残ります。

    
←B受け取る（PC→スマホ）。AIに [[secret:名前]]値[[/secret]] で囲んで返すよう頼みます。例:「生成したトークンを [[secret:token]]…[[/secret]] で囲んで返して」。実値は金庫チャネルへ分離され、本文には [[secret:token]] だけ。スマホで開錠時のみ表示。

    
💡上の … 部分はタップでコピーできます。値入力欄の🎤で声で値を入れることもできます（声もチャットに残りません）。

    
🎤音声入力（日本語）。話した内容が入力欄に入ります。

    
⚡今すぐ割り込み（実行中だけ表示）。今の生成を止めて、会話はそのまま（文脈を保持）新しい指示で立て直します。「説明が足りなかった／やっぱ違う」時に。

    
✕中断。実行中だけ表示され、作業を全部止めます（新しい指示は出しません）。

    
↑送信。指示を送ります（Enterは改行・誤送信防止）。実行中に押すと「終わってから実行」（待ち行列に追加し、今のが終わってから続けて実行）。
